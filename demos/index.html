3D Research Renderer — Rendering Approaches

Same target — a Transformer architecture — rendered three ways. Open each, orbit/zoom, and pick which one your research map should be built on.
Approach 1recommended

Structured Scene-Graph

AI emits a JSON spec of typed nodes + edges; a renderer draws clean labeled primitives.

+ editable, diffable, notes anchor to node ids
– bounded to your primitive vocabulary

 Approach 2

Generated Three.js Code

AI writes raw Three.js — live attention field, flowing pulses, bloom.

+ unlimited custom visuals & motion
– hard to edit safely / anchor notes

 Approach 4bbycroft-style

Tensor Walkthrough (structure + flow)

Every tensor is a grid of cells; a thread of light flows through the compute. Mimics bbycroft.net/llm.

+ structure of #1 with the motion of #2
– denser to author

 Approach 3

Text-to-3D Meshes

A text-to-3D model returns sculpted, displaced geometry per component.

+ rich organic hero visuals
– blobs aren't diagrams; no stable parts

Nodes ↔ Obsidian Notes (docked)

Click a node → its .md note docks beside the 3D view (resizable + fullscreen). Real vault via File System Access API, wiki-links, voice hook.
